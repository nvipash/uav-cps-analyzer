# UAV-CPS-Analyzer — Документація українською мовою

## Огляд

UAV-CPS-Analyzer — комплексний програмний продукт для моделювання та аналізу кіберфізичних систем безпілотних літальних апаратів (БПЛА) в умовах електромагнітного глушіння. Поєднує класичні моделі радіорозповсюдження, формальну статистичну методологію, методи штучного інтелекту/машинного навчання, координацію множинних глушників, аналіз сценаріїв роїв БПЛА та змагальну ко-еволюцію в єдиний інтегрований інструмент.

**Автори:** Новицький П.С., Степаняк М.В., Національний університет "Львівська політехніка", 2025-2026
**Версія:** 1.2.0
**Ліцензія:** GPL-3.0
**Мова:** Python 3.9+

## Архітектура

Система складається з трьох рівнів:

### Рівень 1: Фізична основа (5 модулів)

- **propagation_models.py** — Радіорозповсюдження: Фрііс, COST 231-Hata, Райс-завмирання, тіньове завмирання, ймовірнісна модель Al-Hourani A2G, BER/PER м'яка деградація, ефекти Доплера, атмосферне затухання (ITU-R P.676), урбаністична корекція multi-path, діаграми спрямованості антен (Cosine), розподіли з важкими хвостами (Студентова t, суміш)
- **fhss_emulator.py** — Емуляція протоколу FHSS: послідовності стрибків на основі LFSR, що імітують поведінку DJI OcuSync 2.0/3.0/3+; 5 стратегій глушіння (широкосмугова, вузькосмугова, свіп, слідкуюча, протокольно-обізнана)
- **monte_carlo_engine.py** — Паралельна Монте-Карло симуляція з bootstrap довірчими інтервалами на межах CI, формальним критерієм збіжності (5% відносна напівширина), QMC (Соболь) та антитетична варіансія, просторова кореляція (модель Гудмундсона), відтворюваність через фіксовані зерна
- **cps_analyzer.py** — Аналіз кіберфізичних систем: сенсорна фузія Демпстера-Шефера (РЧ/Радар/Акустика/ЕО-ІЧ), C-UAS архітектура, Парето-оптимізація вартості/ефективності з мульти-обмеженнями, ранжування за стохастичним домінуванням
- **config.py** — Конфігурація: база дронів (DJI Mavic 3, Mini 4 Pro, FPV, волоконно-оптичний), специфікації глушників, 5 пресетів середовищ (щільна_міська, міська, приміська, сільська, відкрите_поле)

### Рівень 2: Статистика та валідація (3 модулі)

- **sensitivity.py** — Глобальний аналіз чутливості: індекси Соболя (першого порядку S1, повного порядку ST), скринінг елементарних ефектів за методом Морріса, паралелізовано через multiprocessing
- **validation.py** — Фреймворк ASME V&V 20: 48 валідаційних випадків з 6 верифікованих джерел (Adamy 2015, Poisel 2011, Skolnik 2008, FCC ID, C-UAS огляди/ITU-R P.1411, Khawaja 2019 A2G польові виміри); MAPE розбивка по доменах (близький/середній/далекий діапазон, регуляторний, field_measurement, field_meas_estimated); 7 перевірок внутрішньої узгодженості
- **reporting.py** — Генерація LaTeX-таблиць (формат booktabs), Markdown зведені звіти з часовими мітками та метаданими відтворюваності

### Рівень 3: AI/ML та розширені (11 модулів — 8 справжніх AI/ML, 3 симуляція/візуалізація)

- **ai_surrogate.py** — Сурогатна модель на основі нейронної мережі (MLP/GP/ансамбль), навчена на виходах Монте-Карло; досягає R²≥0.96 з 200 точками тренування та прискоренням 2000-3000× *(метрика на синтетичних тестових даних)*
- **ai_optimizer.py** — Байєсівська оптимізація з Gaussian Process сурогатом (ядро Матерна 5/2) та функцією придбання Expected Improvement; початкова вибірка Latin Hypercube, максимізація EI через L-BFGS-B з 20 випадковими перезапусками; оптимізація сенсорних комплексів з бюджетними обмеженнями
- **ai_propagation_correction.py** — ML-корекція моделі розповсюдження: Gradient Boosting на залишках з 20+ референсних точок
- **ai_adaptive_jamming.py** — Агент навчання з підкріпленням (Q-learning) для вибору стратегії глушіння; підтримує адверсаріальне середовище та мульти-цільову винагороду (jam_rate + power_efficiency + stealth)
- **ai_threat_classifier.py** — Ансамблева НМ (MLP + Random Forest + Gradient Boosting) для класифікації загроз; 24-параметричний вхід включаючи часові ряди, спектральні, поведінкові ознаки; досягає 99.6% точності проти 32% для чистого Демпстера-Шефера *(метрика на синтетичних тестових даних)*
- **ai_uncertainty_active.py** — Multi-output поширення невизначеності зі справжнім поліноміальним хаос-розкладанням (поліноми Ерміта, порядок 3); цикл активного навчання з GP uncertainty sampling
- **multi_jammer_coordination.py** *(нове)* — Координовані мережі множинних глушників: комбінація потужностей у цілі, теплові карти покриття; повноцінний генетичний алгоритм оптимізації розміщення (елітизм + турнірна селекція + арифметичний blend crossover + мутація)
- **swarm_scenarios.py** *(нове)* — Атаки роїв БПЛА: 4 типи (кооперативний, децентралізований, камікадзе, обманка+удар); аналіз кривої насичення проти ємності мережі глушників
- **ai_coevolution.py** *(нове)* — Адверсаріальне AI-vs-AI навчання: одночасне Q-learning глушника та FHSS-захисника; оцінка рівноваги Неша; виявлення циклів
- **trajectory_scenarios.py** *(нове)* — Часо-залежні траєкторії БПЛА: лінійний підхід, проліт, кругова орбіта, ухильний Безьє; еволюція J/S у часі
- **visualization.py** — Публікаційно-якісні фігури (5 графіків, 300 DPI, PDF + PNG)

## Базові модулі

### `propagation_models.py`

Реалізує 13 моделей розповсюдження/РЧ:

| Модель | Стандарт | Діапазон |
|--------|----------|----------|
| FriisModel | ITU-R P.525-4 | Вільний простір, всі дистанції |
| COST231HataModel | COST 231 | Міські/приміські, 1.5-2.4 ГГц |
| RiceFadingModel | Rice 1944 | Багатопроменеве LOS |
| ShadowFading | Логнормальне/Студентова-t/суміш | Великомасштабне завмирання |
| AltitudeDependentModel | Лінійне змішування | Комбінована міська/вільний простір |
| AlHouraniA2GModel | Al-Hourani 2014 | Ймовірнісна A2G з ймовірністю LOS |
| BERModel | Proakis 2007 | BPSK/QPSK/16-QAM/64-QAM |
| DopplerModel | Класична | Частотний зсув, час когерентності |
| AtmosphericAbsorption | ITU-R P.676 | Кисень + водяна пара |
| UrbanMultiPathCorrection | Емпірична | NLOS додаткові втрати |
| ModulationType | Enum | Схеми модуляції |
| CosinePattern | Косинус | Спрямовані антени |
| OmnidirectionalPattern | Константа | Всеспрямовані антени |

### `fhss_emulator.py`

Параметри емуляції FHSS:
- 40 каналів у діапазоні 2.4 ГГц (2400-2483.5 МГц)
- Ширина каналу: 2 МГц (2.5 МГц для OcuSync 3+)
- Швидкість стрибків: 500 Гц (час перебування 2 мс)
- LFSR-16 з тапами максимальної довжини [16, 15, 13, 4]

Зверніть увагу: це поведінкова емуляція, заснована на публічних регуляторних даних (FCC ID) та специфікаціях виробника (DJI). Реальний протокол OcuSync є пропрієтарним. Посилання "Beason et al. (2021)" та "Schiller et al. (2023)", присутні у попередніх версіях, не підтверджені як реальні публікації і були видалені.

### `monte_carlo_engine.py`

Три методи семплювання:
- **Стандартна МК** (`run_simulation`) — базове псевдовипадкове
- **QMC (Соболь)** (`run_simulation_qmc`) — квазі-випадкове з низькою дискрепантністю для кращої збіжності
- **Антитетичні варіансії** (`run_simulation_antithetic`) — парне семплювання для зменшення варіансії

Адаптивна збіжність (`run_simulation_adaptive`) автоматично зупиняється, коли відносна напівширина CI падає нижче порогу (за замовчуванням 5%).

## Модулі AI/ML

### Сурогатна модель (`ai_surrogate.py`)

Тренує нейронну мережу на виходах Монте-Карло для миттєвого передбачення J/S:

```python
from ai_surrogate import SurrogateTrainer, SurrogatePredictor

trainer = SurrogateTrainer(mc_iterations_per_point=500)
X, Y = trainer.generate_training_data(n_points=200)
model, sx, sy, metrics = trainer.train(X, Y, model_type='ensemble')
predictor = SurrogatePredictor(model, sx, sy)
result = predictor.predict(power_dbm=40, distance=500, ...)
# Результат за <1мс проти 1000мс для повного MC
```

### Класифікатор загроз (`ai_threat_classifier.py`)

24-параметричний ансамблевий класифікатор:
- Класичні сенсорні ознаки (10): РЧ, Радар, Акустика, ЕО/ІЧ детекції + параметри
- Поведінкові ознаки (6): варіації RCS, швидкості, висоти, курсу, плавність польоту, реакція на глушіння
- Спектральні РЧ ознаки (5): ширина смуги, складність модуляції, швидкість стрибків, бурстність, спектральна ексцес
- Акустичні спектральні ознаки (3): фундаментальна частота, гармонічне співвідношення, доплерівський зсув

### Ко-еволюція (`ai_coevolution.py`)

Мульти-агентне RL, де глушник і FHSS-захисник навчаються одночасно:
- Дії глушника: 4 стратегії (широкосмугова, вузькосмугова, свіп, слідкуюча)
- Захист FHSS: 5 поз (uniform_hopping, avoid_jammed_channels, high_modulation, spread_pattern, frequency_diversity)
- Гра з нульовою сумою: Q-learning для обох агентів
- Виходи: змішана стратегія рівноваги Неша, динаміка збіжності, виявлення циклів

## Розширені модулі

### Координація множинних глушників

`MultiJammerSimulator` комбінує потужність від кількох глушників у лінійному домені:

```python
from multi_jammer_coordination import (
    MultiJammerSimulator, JammerNetwork, JammerNode, create_default_networks
)

networks = create_default_networks()
sim = MultiJammerSimulator()
coverage = sim.coverage_map(networks['5_mesh'])
# Повертає 2D теплову карту P(jam) на території 6км×6км
```

Передвизначені мережі:
- `1_portable` — один 10Вт глушник (базовий)
- `3_triangle` — 3 глушники у трикутнику (периметрова оборона)
- `5_mesh` — мережа з 5 глушників (центральний + 4 спрямованих)
- `2_stationary` — 2 високопотужні фазовані решітки

`JammerNetworkOptimizer` використовує генетичний алгоритм для пошуку оптимального розміщення.

### Сценарії роїв

Реалізовано 4 типи роїв:
- **COOPERATIVE** — координований, вимагає спільного зв'язку; заглушений → втрачений
- **DECENTRALIZED** — незалежні агенти, більш стійкі
- **KAMIKAZE** — автономні, важче відключити
- **DECOY_STRIKE** — обманки поглинають глушіння для справжніх атакуючих

```python
from swarm_scenarios import SwarmAttackSimulator, SwarmConfig, SwarmType

config = SwarmConfig(swarm_type=SwarmType.COOPERATIVE, n_uavs=20)
sim = SwarmAttackSimulator()
result = sim.simulate_attack(swarm, jammer_network)
# Повертає рівень виживання, час до прориву, пік активних, прапор насичення
```

## Фреймворк валідації

Методологія **ASME V&V 20**:
- E = |передбачення_моделі - референсне_значення|
- u_val = sqrt(u_model² + u_exp²)
- PASS якщо |E| < u_val

48 валідаційних випадків у 6 доменах (реальні результати, v1.2):
- close_range (5 випадків): MAPE ~24%, PASS 0% — модель недооцінює втрати на дуже малих відстанях
- medium_range (17 випадків): MAPE **25.8%**, PASS **71%** — найкраще відкалібрований домен
- long_range (9 випадків): MAPE ~45%, PASS 14% — обмеження моделі поширення понад 5 км
- regulatory (5 випадків): MAPE ~98% — еталони FCC припускають ближче джерело сигналу
- field_measurement (5 випадків): A2G Khawaja (2019) — міський, інтерпольовані з виміряних exponents
- field_meas_estimated (7 випадків): A2G Khawaja (2019) — приміський/сільський, фізично оцінені exponents (слабша верифікація)

> **Щодо параметрів сигналу:** кейси Adamy/Poisel/Skolnik припускають тактичне радіо
> (P = 10 W, G = 0 дБі). FCC/UAV кейси — UAV-пульт (P = 20 dBm, G = 2 дБі).
> Деталі: `_make_case()` у `validation.py`.
>
> **Щодо вилученої GROUP E:** FHSS/OcuSync behavioral-кейси прибрано — верифікованих
> відкритих вимірювань для OcuSync не існує (див. `literature_dataset.py`).
>
> **Щодо ймовірностей виявлення сенсорів (cps_analyzer.py):** значення Pd=0.99/0.85/0.95/0.80
> та PREDICTION_ACCURACY=0.85 — інженерні оцінки без верифікованих джерел.
>
> **Щодо AI/ML модулів:** всі навчальні дані синтетичні (Monte Carlo).
> Метрики точності відображають якість апроксимації симуляції, а не фізичну
> достовірність. Для оцінки фізичної точності — дивіться результати валідації вище.

## Посібник з розширення

Щоб додати нову модель розповсюдження:
1. Успадкувати від `PropagationModel` ABC у `propagation_models.py`
2. Реалізувати метод `path_loss(distance_m, **kwargs)`
3. Додати до factory function `get_model()`
4. Додати валідаційні випадки у `validation.py`

Щоб додати новий валідаційний випадок:
1. Додати кортеж до `_define_validation_cases()` у `validation.py`
2. Вказати поле `domain` для аналізу по доменах
3. Використовувати helper `_make_case()`

Щоб додати новий AI модуль:
1. Створити файл `ai_<name>.py`
2. Експортувати класи через `__init__.py`
3. Додати крок у `run_analysis.py`
